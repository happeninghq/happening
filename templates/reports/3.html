June 2014 Dojo

This month we decided to ban the use of Python incase we inadvertantly became the Southampton Python Dojo. Instead we voted on Facebook and decided that we'd use Javascript.

The Challenge

This month's challenge was more open than usual - and involved implementing the game Mastermind, for either one or two players. Everything else was left up to the participants.

Solutions

Group 1 did not complete, having split up into too many subgroups, and was unable to bring the whole thing together in the end. Afterwards, one team member did successfully implement Donald Knuth's five guess algorithm but this wasn't completed in time to be demoed on the night.

All other groups managed to produce a version of Mastermind where the user can guess the code selected by the computer. Some groups used HTML tables for the UI, with CSS border-radius for the pegs, and at least one used canvas to draw the interface.

This group created a version of the game using tables for layout with border-radius to represent pegs, where the player can click the pegs to make their guess.

If any other groups have their code to submit, please email me.
